Claude Code

Best Practices for Agentic Coding

Leveraging the Power of CLI

What is Claude Code?

  • A command-line tool for agentic coding by Anthropic.
  • Designed to be low-level and unopinionated.
  • Offers engineers flexibility and customization.
  • Enables programmatic interaction with large language models for code generation and manipulation.

Benefits of CLI Thinking

Why a Command-Line Interface Approach?

  • Automation: Easily script and integrate into existing workflows.
  • Control & Precision: Fine-grained control over inputs and outputs.
  • Flexibility: Adapt to diverse tasks without GUI limitations.
  • Agentic Capabilities: Ideal for building intelligent agents that interact with code.
  • Reproducibility: Commands can be easily logged and re-run.

Best Practice: Customizing Setup

  • Tune `CLAUDE.md` files:
    • Provide essential context for Claude.
    • Guide its behavior and understanding of the codebase.
    • Example: Describe project structure, coding conventions, or specific library usages.
  • Curate Claude's Tools:
    • Selectively allow tools for safety and efficiency.
    • Prevent unintended actions or access.
    • Example: Restrict file deletion or network access if not required for the task.

Best Practice: Expanding Claude's Tools

  • Leverage Bash Tools:
    • Integrate standard command-line utilities.
    • Example: Use `grep -r "TODO"` to find all TODOs in the codebase, or `git diff` to review changes.
  • Integrate with company CLI tools or MCP Servers:
    • Connect to internal services, data cli tools, and APIs.
    • Example: Allow Claude to query an internal logging service or deploy a test build.

Common Workflows: Development

  • Explore, Plan, Code, Commit:
    • For complex problem-solving.
    • Structured approach for new features/bugs.
    • Example: "Explore the `auth` module, plan how to add OAuth, then implement and commit."
  • Write Tests, Commit; Code, Iterate, Commit:
    • Test-Driven Development (TDD) with Claude.
    • Ensures correctness and regression prevention.
    • Example: "Write a failing test for the new `calculate_discount` function, commit it, then write the function until the test passes."

Common Workflows: Beyond Coding

  • Write Code, Screenshot Result, Iterate:
    • Ideal for visual tasks (UI/UX).
    • Claude gets visual feedback.
    • Example: "Generate CSS for a button, I'll provide a screenshot, then refine based on visual feedback."
  • Codebase Q&A and GitHub Interactions:
    • Querying code for understanding.

Optimizing Workflows: Instructions

  • Be Specific in Instructions:
    • Clear, unambiguous prompts yield better results.
  • Provide Images and URLs:
    • Visual context is powerful for Claude.
    • Point to relevant documentation or examples.
    • Example: "Make the button look like the one at `https://example.com/design-system/button`" or "Here's a screenshot of the current UI, make the text bold."
  • Course-Correct Early and Often:
    • Don't let Claude stray too far.
    • Example: If Claude starts modifying unrelated files, immediately stop and redirect it to the correct scope.

Conclusion

Mastering Claude Code for Efficient Agentic Coding

  • Customize your setup for optimal context.
  • Expand Claude's capabilities with more tools.
  • Adopt effective workflows for various tasks.
  • Optimize interactions with clear instructions and feedback.
  • Explore advanced techniques for automation and collaboration.
